## 1. Paper Selection

The group discussed candidate papers and agreed to proceed with the **Monks et al. paper**.

### Rationale for Selection:

* Strong reproducibility potential
* Clear model structure and pathway logic
* Sufficient reporting of parameters and process flow
* Availability of published data to support validation
* Appropriate complexity for HPDM097 requirements

The group agreed that this paper provides a suitable balance between technical feasibility in SimPym, sufficient but not excessive model complexity, opportunity for critical analysis and validation, scope for experimentation.

---

## 2. Development Approach

The group agreed to adopt:

### 1. LLM-Assisted Development

* Use LLMs to support:

  * Extraction of model logic
  * Parameter identification
  * Drafting documentation
  * Iterative code refinement
* Critically evaluate LLM outputs rather than accepting them uncritically.
* Document prompt engineering and refinement process for inclusion in the report.

### 2. Iterative Development Strategy

The model will be built incrementally:

* Start with simplified core process logic
* Add complexity progressively
* Test and validate each stage before moving forward
* Maintain version control and clear documentation of iterations

This aligns with the assignment guidance on iterative recreation and testing.
